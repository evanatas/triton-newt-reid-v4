## Что это
Re-identification тритонов по узору пятен **брюшка** методом **локального признакового матчинга** (не global-эмбеддинг).
Версия 3.0 (global MegaDescriptor) провалилась: финал top-1 **0.188**. Версия 4.0 — разворот на local-features.

## Пайплайн (5 шагов)
1. **Сегментация** — BiRefNet matting тела → кроп 512×512, нормализация масштаба по высоте маски, метка-номер маскируется
   (анти-утечка). Код: `segment/build_crops.py` → `segment/crops/` + `crops_manifest.csv`.
2. **Препроцессинг** — RGB → gray → **CLAHE(clip=2.0, 8×8)**. Единый для gallery и probe (урок H1: разный препроцессинг врёт).
3. **Матчер** — **SIFT** (nfeatures=1500) + BFMatcher L2 knn + **Lowe ratio 0.75**. `core/matchers.py:SiftMatcher`.
4. **Деформ-верификация** — **`cv2.estimateAffine2D` (6-DOF affine)**, RANSAC thr=5.0. score = число affine-inlier.
   `core/deform.py:verify(..., "affine")`. Допускает анизотропное растяжение узора у растущих особей — главный рычаг на 2-мес интервале.
5. **Галерея** — **rolling multi-reference**: для probe из сессии T референсы = все кадры особей из сессий **строго раньше** T;
   score особи = max-pool. Прод-режим (заказчик даёт ежемесячные съёмки). `core/eval_core.py`.
