# Примеры фото для проверки демо

Загружайте эти снимки во вкладке **«🔎 Опознание» → «Загрузить фото»** (демо: `streamlit run backend/demo.py`, http://localhost:8502).

Демо **честное**: при опознании из каталога исключается **вся съёмочная сессия запроса** (тот же день),
поэтому узнавание идёт по снимкам **других месяцев** — это реальный temporal re-id, а не совпадение дубликата.

## Что показывает каждый снимок

| Файл | Ожидаемый вердикт | Что демонстрирует |
|---|---|---|
| `01_известная_temporal_TK-031.JPG` | ✅ известна (~84 %) | узнавание особи по снимку **другого месяца** (temporal re-id) |
| `TK-006_2мес_*`, `TK-011_2мес_*`, `TK-012_3мес_*` | 🟡 на проверку | top-1 верный, уверенность умеренная — human-in-the-loop |
| `02_новая_TK-062_снимок1.JPG` | 🔴 новая | особи нет в базе (open-set) |
| `03_новая_TK-062_снимок2.JPG` | 🔴 новая → ✅ после регистрации | второй снимок той же новой особи |

## Демо флоу учёта новых особей
1. Загрузите **`02_новая_TK-062_снимок1.JPG`** → вердикт **«🔴 новая»**.
2. Разверните **«➕ Зарегистрировать эту особь в базе»** → задайте имя (напр. «Спот-1») → **«Зарегистрировать»**.
3. Загрузите **`03_новая_TK-062_снимок2.JPG`** → теперь система **узнаёт её** как «Спот-1».

Так работает учёт: **новая особь → регистрация → при следующей поимке узнаётся**.
В демо особь добавляется в каталог на время сессии; в продакшене регистрация пишется в БД (SQLite→PostgreSQL).
